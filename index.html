03
Тиждень третій
Покажчики та динамічна пам'ять
~10 днів ⭐⭐⭐⭐ СКЛАДНО
⚠️ Найважча тема. Без покажчиків BST написати неможливо — кожен вузол дерева з'єднаний з іншими саме через покажчики. Не поспішай, повторюй на малих прикладах.
🎯 Основи покажчиків
  • Що таке адреса змінної в пам'яті
  • int *p — оголошення покажчика
  • &x — взяти адресу змінної x
  • *p — отримати значення за адресою
  • NULL — порожній покажчик (немає вузла)
🔗 Покажчики на struct
  • Node *p — покажчик на вузол дерева
  • p->name замість (*p).name
  • Node **root — покажчик на корінь дерева
  • Передача struct у функцію через покажчик
🧱 malloc та free
  • malloc(sizeof(Node)) — виділити пам'ять для вузла
  • free(p) — звільнити пам'ять після видалення
  • Перевірка: if (p == NULL) після malloc
  • Правило: кожен malloc → обов'язково є free
🔄 Рекурсія
  • Функція що викликає сама себе
  • База рекурсії — умова зупинки (if root == NULL)
  • Рекурсивний крок — виклик для лівого/правого сина
  • Всі операції BST написані рекурсивно

Напиши функцію Node* createNode(User data) що через malloc виділяє пам'ять для нового вузла, записує дані і ставить left = right = NULL. Напиши рекурсивну freeTree(Node *root) що звільняє всі вузли.

04
Тиждень четвертий
Binary Search Tree (BST)
~14 днів ⭐⭐⭐⭐⭐ НАЙСКЛАДНІШЕ
🌳 Це ядро всієї програми. Не переходь далі поки не напишеш і не протестуєш всі 4 операції. Відкрий visualgo.net/en/bst щоб бачити як це виглядає.
🏗️ Структура вузла BST
  • typedef struct Node { User data; struct Node *left; struct Node *right; } Node;
  • Ліве піддерево — менші значення
  • Праве піддерево — більші значення
  • Сортується за іменем через strcmp
➕ Insert — додати користувача
  • Якщо root == NULL — це і є місце для вузла
  • strcmp(name, root->data.name) < 0 → йдемо ліво
  • strcmp(name, root->data.name) > 0 → йдемо право
  • Рекурсія до досягнення NULL
🔍 Search — пошук
  • Той самий принцип що й Insert
  • Повертаємо Node* якщо знайшли, NULL якщо ні
  • O(log n) — набагато швидше ніж перебір
  • Пошук за іменем для User, за назвою для Book
📋 Inorder — вивести всіх
  • Ліво → Вузол → Право
  • Дає відсортований список — алфавітний порядок
  • Рекурсивна функція з 3 рядків коду
  • Використовується для "показати всіх користувачів"
🗑️ Delete — видалити
  • Випадок 1: вузол — лист → просто видалити
  • Випадок 2: один нащадок → замінити нащадком
  • Випадок 3: два нащадки → знайти мінімум правого піддерева
  • Inorder successor — найменший у правому піддереві

Напиши BST спочатку для цілих чисел (простіше): insert(), search(), inorder(), delete(). Тест: вставити 5, 3, 7, 1, 4 — inorder має вивести 1 3 4 5 7. Потім перепиши для User де ключ — name через strcmp.

05
Тиждень п'ятий
Збереження у файл та завантаження
~7 днів ⭐⭐⭐ СЕРЕДНЬО
📂 Основи роботи з файлами
  • FILE *f = fopen("users.dat", "w") — відкрити для запису
  • fopen("users.dat", "r") — відкрити для читання
  • fclose(f) — закрити файл (завжди!)
  • if (f == NULL) — перевірка що файл відкрився
✍️ Запис і читання
  • fprintf(f, "%d %s %s\n", id, name, email) — записати рядок
  • fscanf(f, "%d %s %s", &id, name, email) — зчитати рядок
  • feof(f) — чи досягнуто кінець файлу
  • Формат: 1 запис = 1 рядок файлу
💾 Серіалізація BST (ключова техніка)
  • Preorder обхід для збереження дерева у файл
  • Preorder зберігає форму — при завантаженні дерево буде таке саме
  • Inorder НЕ підходить — дерево відновиться незбалансованим
  • Завантаження = читати файл + викликати insert() для кожного рядка

Напиши saveUsers(Node *root, FILE *f) — Preorder обхід, записує кожного користувача через fprintf. Напиши loadUsers(FILE *f, Node **root) — читає рядки і викликає insert(). Перевір: зберегти → вийти → запустити знову → завантажити → inorder дає той самий результат.

06
Тиждень шостий
Редагування та пошук
~7 днів ⭐⭐⭐ СЕРЕДНЬО
✏️ Редагування запису
  • Редагування в BST = delete старого + insert нового
  • Знайти вузол через search()
  • Зчитати нові дані від користувача
  • Видалити старий вузол, вставити новий
🔍 Пошук за різними полями
  • BST за іменем — швидкий пошук O(log n)
  • Пошук за id — обхід всього дерева O(n)
  • Пошук книги за автором — обхід з перевіркою
  • Inorder + фільтр — для пошуку за будь-яким полем
🖥️ Підменю для кожного розділу
  • Підменю Користувачі: 1.Додати 2.Знайти 3.Редагувати 4.Видалити 5.Всі 0.Назад
  • Підменю Книги: аналогічно
  • Валідація вводу — що робити якщо введено не число
  • system("cls") — очистити консоль між діями

Додай до програми повноцінне підменю для користувачів з усіма операціями. Переконайся що після кожної дії (додати, видалити, знайти) програма повертається до підменю і не падає на некоректному вводі.

07
Тиждень сьомий
Збираємо повну програму
~10 днів ⭐⭐⭐⭐ СКЛАДНО
🗂️ Розбити код на файли
  • user.h / user.c — BST для користувачів
  • book.h / book.c — BST для книг
  • main.c — тільки меню і виклики функцій
  • Header guards: #ifndef USER_H / #define USER_H / #endif
  • #include "user.h" у main.c
📚 Модуль книг
  • Окреме BST для книг, ключ — title через strcmp
  • Всі ті ж операції: insert, search, delete, inorder
  • Пошук за автором — обхід inorder з перевіркою strcmp
  • saveBooks() / loadBooks() у файл books.dat
✅ Фінальна перевірка
  • Дані зберігаються в users.dat і books.dat
  • Після перезапуску програми — дані відновлюються
  • Всі операції CRUD працюють для обох розділів
  • Програма не падає на некоректний ввід
  • Немає витоків пам'яті (free для всього)

З'єднай всі модулі. Фінальне тестування: 1) додай 5 користувачів і 5 книг; 2) закрий програму; 3) запусти знову — всі дані мають завантажитись; 4) знайди, відредагуй, видали запис; 5) знову збережи і перезапусти.

🏁 Готово! Якщо програма проходить це тестування — ти написав повноцінну систему з BST, файлами та CRUD. Це серйозний результат.
